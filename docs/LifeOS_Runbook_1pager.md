# LifeOS 1页运行手册
更新时间：2026-03-09

## 1. 你每天只做这件事
1. 打开网站，填 `DL-30`（30-45秒）
2. 提交后拿到 `A/B/C/D`
3. 只执行 `B`（<=15分钟；红灯日<=5分钟）

## 2. DL-30 填写最小模板
```text
DL-30 | YYYY-MM-DD
行为：____（动词+对象+时长/次数）
结果：____（完成/产出/指标）
昨日B执行：0/1
RUIN：0/1（若1：码=S/T/P）
明日意图：____（可空）
```

## 3. 关键规则（写死）
1. 不断更优先：忙爆也至少提交 Zero-Day。
2. 只做一个最小行动 B：不要并行开新战线。
3. 明日意图为空：默认 B 走 Output > Training > Input control。
4. RUIN=1：自动进入防崩模式，B <= 5 分钟。

## 4. 红灯码速记
1. `S`：睡眠红灯
2. `T`：交易违规
3. `P`：关键任务掉线

## 5. 每周（10分钟）
打开 `WR-10` 折叠区，完成：
1. 本周最有效动作（1条）
2. 本周最大失控（1条）
3. if-then（1条）
4. 下周唯一优先事项（1条）
5. 下周唯一实验（7天，<=15分钟/天）
6. 下周唯一改动（只改1条）

## 6. 每两周（5-10分钟）
打开 `System Patch`：
1. 写卡点
2. 写断点
3. 只改一处
4. 生成新规则（<=3条）

## 7. 每月（10分钟）
打开 `Playbook 月度升级`：
1. 生成 Top动作/Top触发器
2. 固化 Anti-ruin rules
3. 更新实验库

## 8. Dojo（随时5分钟）
打开 `Dojo`，填写 `场景/目标/约束`，生成 5 轮训练脚本。

## 9. Zero-Day（忙爆不断更）
```text
DL-30 | YYYY-MM-DD
行为：____（<=5分钟）
结果：____
昨日B执行：0/1
RUIN：0/1
明日意图：
```

## 10. 一句话原则
降级不算失败，断更才算失败。
