## 1. 产品定义
### 1.1 产品一句话
LifeOS Web App 是一个以低摩擦输入、Anti-ruin 防崩、可控变量追踪、AI 输出闭环为核心的个人 AI OS 前端。

### 1.2 产品目标
系统必须同时满足四件事：
1. 长期不崩：低状态也能完成最小输入，不断更。
2. 单位时间产出最大：每次只输出一个高杠杆最小行动 B。
3. 可持续变强：通过 B 执行、红灯天数、周实验持续迭代。
4. 低摩擦闭环：DL-30 -> A/B/C/D -> 执行 B -> 次日追踪 B -> WR-10 -> Patch -> Playbook。

### 1.3 产品不做什么
本产品不是情绪陪伴 App，不是复杂记账工具，不是大而全任务管理器，不是社交平台。

## 2. 核心指标
### 2.1 北极星指标
7 日持续使用率 + B 执行率。

### 2.2 关键观测
1. DL 完成天数（7 日）
2. B 执行天数（7 日）
3. 红灯天数（7 日）
4. 周实验完成率

## 3. 用户与场景
### 3.1 目标用户
高自我要求、重效率与纪律、需要在多任务压力下维持长期复利的个人用户。

### 3.2 高频场景
1. 晚间 30 秒提交 DL-30
2. 早间执行昨日 B
3. 每周 10 分钟 WR-10
4. 连续失败时做 System Patch
5. 每月生成 Playbook
6. 白天 5 分钟 Dojo 训练

## 4. 产品原则
### 4.1 不可谈判原则
1. 单入口优先：默认首页围绕 DL-30。
2. 不断更优先：允许降级输入，不允许因复杂度断更。
3. 控制变量优先：B 执行比记录细节更重要。
4. Anti-ruin 优先：红灯先防崩，再谈增益。
5. 默认化优先：疲劳状态下系统替用户做默认决策。

### 4.2 交互原则
默认折叠低频模块，默认给最小动作，默认减少认知切换，默认支持导出迁移。

## 5. 版本范围
### 5.1 v1 必须包含
1. DL-30 主入口
2. 昨日 B 执行 0/1
3. RUIN 0/1 + S/T/P
4. 红灯自动降级（B <= 5 分钟）
5. 默认 B 规则
6. A/B/C/D 输出复制
7. WR-10 + Scoreboard-5
8. System Patch
9. Playbook 月度沉淀入口
10. 本地存储
11. 数据导出能力

### 5.2 v1 不必须包含
登录系统、多人协作、社交分享、云同步、复杂图表、原生 App。

## 6. 信息架构
### 6.1 页面结构
建议单页结构：
1. DL-30 主区
2. A/B/C/D 输出区
3. 最近 7 天摘要
4. WR-10 折叠区
5. System Patch 折叠区
6. Playbook 折叠区
7. Dojo 折叠区
8. 导出/导入/设置折叠区

### 6.2 默认展开规则
默认只强调 DL-30，其他模块折叠。

## 7. 核心功能需求
### 7.1 DL-30（日常唯一入口）
必填字段：
1. 日期（默认今日）
2. 行为
3. 结果
4. 昨日 B 执行（0/1）
5. RUIN（0/1）
6. RUIN 码（RUIN=1 时必选 S/T/P）
7. 明日意图（可空）

规则：
1. 明日意图为空时，默认 B 优先级 = Output > Training > Input control。
2. 支持一键复制 DL 文本。
3. 支持一键复制 A/B/C/D 文本。

### 7.2 RUIN 与自动防线
RUIN=1 时强制行为：
1. B 自动降级到 <= 5 分钟。
2. C 自动输出对应防线。
3. 计入红灯统计。

码化：
1. S：睡眠红灯
2. T：交易违规
3. P：关键任务掉线

### 7.3 A/B/C/D 输出
系统每次提交必须输出：
1. A：模式判断（1-2句）
2. B：最小行动（步骤化）
3. C：风险点 + 1条防线
4. D：tail bet（可选）

### 7.4 WR-10（每周）
字段：
1. 本周最有效动作
2. 本周最大失控
3. if-then 模式
4. 下周唯一优先事项
5. 下周唯一实验
6. 下周唯一改动
7. Scoreboard-5

功能：
1. 可读取近 7 天 DL 数据自动预填
2. 可生成并复制 WR-10

### 7.5 System Patch（双周）
触发意图：断更、连续失败、红灯偏高时只改一处。

输出：
1. 系统卡点
2. 最常断点
3. 减摩擦改动
4. 新规则（<=3条）

### 7.6 Playbook（月度）
生成结构：
1. Top 高收益动作
2. Top 高频触发器
3. Anti-ruin rules
4. 实验库

### 7.7 Dojo（训练）
输入：场景、目标、约束。  
输出：5轮训练脚本 + 评分维度 + 复盘方式。

## 8. 数据需求
### 8.1 数据对象
DailyLog, WeeklyReview, SystemPatch, Playbook。

### 8.2 数据策略
1. 默认本地存储优先
2. 支持导出 JSON / CSV / Markdown
3. 支持导入恢复
4. 明确说明是否上传服务端

## 9. 非功能需求
1. 首屏加载 < 2 秒
2. 提交后输出 < 1 秒
3. 手机端可用，关键路径单手可操作
4. 低网/离线情况下尽量可记录
5. 隐私策略明确

## 10. 验收标准（总览）
v1 完成标准：
1. 30 条验收项至少通过 27 条
2. 数据导出可用
3. 红灯自动防线可用
4. B 执行追踪可用
5. 日常入口保持低摩擦
